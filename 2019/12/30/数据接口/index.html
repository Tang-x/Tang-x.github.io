网络API

网络应用程序，分为前端和后端两个部分。前端都是通过一个url去拿到数据，然后将数据渲染到页面上。当前后拿给前端分离开发的时候，后端很重要的一个工作就是写好各种API接口，将写好的接口提供给前端，然后前端再通过这个接口去获取JSON数据。

1.什么是API

  • API - 是一些预先定义的函数，目的是提供应用程序与开发人员基于某软件或硬件得以访问一组例程的能力，而又无需访问源码，或理解内部工作机制的细节
  • API - 先把这些复杂的操作写在一个函数里面，编译成一个组件（一般是动态链接库），随操作系统一起发布，并配上说明文档，程序员只需要简单地调用这些函数就可以完成复杂的工作，让编程变得简单有趣(
  • 通俗来说，别人写好的代码，或者编译好的程序，提供给你使用，就叫做API

2.本次主要讲的是网络API – RESTful API

RESTful是一种架构的规范与约束、原则，符合这种规范的架构就是RESTful架构。

在实际的使用过程中要围绕两件事来做：无状态和幂等性

无状态：是指每次的请求都是独立的，它的执行情况和结果与前面的请求和之后的请求是无直接关系的。

幂等性：是指不管你调用一次，还是调用一百次，一千次，结果都是相同的。
